# Glossary

Use these patterns when a product teardown mentions jargon that a non-technical reader may not know.

Keep the explanation short:
- one plain-language sentence
- one analogy if helpful
- then continue with the real term

## Web and app basics

- frontend: the part the user directly sees and clicks
- backend: the behind-the-counter part that handles logic and coordination
- API: a structured way for one piece of software to ask another piece for something
- webhook: an automatic heads-up message one service sends to another when something happens
- authentication: the login check that confirms who you are
- authorization: the permission check that decides what you are allowed to do
- database: the system's organized memory cabinet
- cache: a short-term holding area used to speed repeated lookups
- queue: a waiting line for jobs that should run in the background
- worker: a background helper that picks up jobs from the queue

## File, media, and content pipeline terms

- object storage: a warehouse for large files such as images, videos, and exports
- CDN: a network of nearby delivery points that helps files load faster
- transcoding: converting media from one format or quality level to another
- codec: the rulebook that says how audio or video gets compressed and unpacked
- container: the outer box that holds video, audio, captions, or metadata together
- bitrate: how much data is used each second to represent the media
- fps: how many still pictures are shown every second to create motion
- resolution: how much visual detail fits into each frame
- muxing: putting separate media tracks into one output package

## AI and automation terms

- inference: the moment an AI model produces an answer from an input
- embedding: a numeric fingerprint that helps software compare meaning or similarity
- vector database: a storage system built for finding similar meanings fast
- retrieval: fetching the most relevant pieces before generating an answer
- agent: a software worker that can decide which step or tool to use next
- orchestration: the traffic control layer that coordinates multiple steps or services

## Billing and SaaS terms

- subscription: a repeating payment for continued access
- entitlement: the list of features or limits a paid user has unlocked
- metered billing: charging based on actual usage instead of a flat plan
- seat-based pricing: charging per team member or account
- free tier: a limited plan designed to let users start without paying

## Infrastructure terms

- serverless: code that runs on demand without the team managing dedicated servers
- containerized service: an app packed with what it needs so it can run consistently
- load balancer: a traffic director that spreads requests across multiple machines
- observability: the logs, metrics, and traces that help operators see what the system is doing
- rate limit: a cap on how often something can be called in a period of time
